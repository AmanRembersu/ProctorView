Overview

ProctorBView is a secure, Electron-based desktop browser built for high-stakes online assessments. It ensures academic integrity by disabling shortcuts, locking the test environment, and preventing screen capture or browser tampering. Ideal for educational institutions and corporate certification bodies, ProctorBView integrates with any web-based assessment system via configuration.

Key Features

  • Full-screen locked kiosk mode to prevent switching apps
  • Disabled right-click, shortcuts (Ctrl, F12, F5, PrintScreen)
  • Predefined secure URL loading from config
  • Electron-powered custom secure browser interface
  • Custom top-bar with date display and controlled exit
  • Iframe-based secure loading of any exam system
  • Daily-updated browser build for enhanced security

Technology Stack

  • Frontend: HTML, CSS (Tailwind), JavaScript
  • Backend: Electron (Node.js)
  • Security Layer: Preload script for input restrictions
  • Configurable: Exam URL stored in JSON

Application Structure

/ProctorBView
├── index.html         # Secure exam interface layout (iframe + UI)
├── main.js            # Electron app launcher with secure config
├── preload.js         # Input restriction (disables keys/menus)
├── config.json        # Target exam form or platform URL
└── package.json       # App metadata and dependencies

Configuration

The exam URL is stored in config.json. Change it to direct users to your own exam portal or form. 

{
  "url": "https://your-secure-exam-link.com"
}

How It Works

  1. Launches in fullscreen kiosk mode (no window controls)
  2. Loads secure exam interface via iframe
  3. Blocks all common shortcuts and developer tools
  4. Shows date and a secure exit button
  5. Prevents screen capture and context menu usage

Installation

# Clone the repo
git clone https://github.com/your-org/proctorbview.git
cd proctorbview

# Install dependencies
npm install

# Start app (dev mode)
npm start

# Build app (for production)
npm run build

Security Measures

  • Right-click disabled via preload.js
  • All keyboard shortcuts like Ctrl, F12, PrintScreen blocked
  • No access to browser dev tools
  • Auto-updated versions with enhanced protection policies
